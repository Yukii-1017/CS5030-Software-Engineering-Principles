Motor Limitations

Mobility disorder including (Brewer, 1999):


  1. Weakness.

  2. Limitations of muscular control (such as involuntary movements, a lack of coordination, or paralysis).

  3. Limitations of sensation.

  4. Joint problems.

  5. Missing limbs.

  6. May include pain, which further hinders movement.

Any of these diseases can affect the hands, arms, and other parts of the body. Those with mobility impairments that affect their hands or arms can use some equipment specifically designed for them: (Brewer, 1999)


  1. “A specialized mouse”

  2. “A keyboard with a layout of keys that matches their range of motion”

  3. “Head-mouse, head-pointer, or mouth-stick”

  4. “Voice-recognition software”

  5. “An eye-gaze system”

  6. “Other assistive technologies”

  7. Can use clicking a single button instead of a combination of multiple buttons at the same time (for example, using "R" instead of "CRTL+R").

Those who have difficulty with hands and arms movement may encounter obstacles when using the web, the obstacles may include: (Brewer, 1999)


  1. “browsers and authoring tools that do not support serialized keystrokes for commands”

  2. “forms that cannot be tabbed through in a logical order”

  3. “interactive forms with time-limited response options”

Tools from Apple

Apple is a hardware and software supplier with expectations. Apple has produced some products designed for people with disabilities to use when browsing websites.


For example, Apple’s Macintosh computer includes screen-magnification software, system software to help people with motion disabilities, electronic documentation, text-to-speech synthesis, voice recognition, and visual alert cues.” (Carter, 2001)

Tools from Microsoft

Microsoft has overlapped a part of the operating system, office efficiency, and browser markets. Therefore, Microsoft's accessibility plan may be the most extensive among all vendors. Its accessibility plan has affected many people and companies. (Carter, 2001)


Microsoft officially approved the accessibility policy called Enable in 1995. This policy has four goals: (Carter, 2001)


  1. “Make accessible products”

  2. “Build strong bonds with the disabled community”

  3. “Equip developers”

  4. “inform customers about the accessibility of products”

For example, the accessibility strategy is embodied in Microsoft's Internet Explorer (IE) product. Compared with the previous version, IE 5 adds some new features that bring convenience to people with mobility impairments . Such as auto completion, auto correction and auto search. These new features can allow users to reduce the use of the keyboard, but also reduce the possibility of input errors. IE 5 also provides a Web attachment toolkit , which allows users to adjust the font and color of text, background and links, use the keyboard to navigate the page, create their own cascading style sheet to view page customization features, and use a screen reader to display ALT text. (Carter, 2001)


Microsoft also provides recommended code examples and checklists for testing the accessibility of web pages. There are many other resources on the Microsoft website, including: (Carter, 2001)


  1. Articles

  2. Step-by-step guides

  3. Descriptions of assistive technologies

  4. Accessible documentation and support

  5. A valuable set of procedures that developers can use to make their products more accessible

Guidelines from the World Wide Web Consortium(W3C)

“World Wide Web Consortium’s Web Accessibility Initiative The World Wide Web Consortium (W3C), an organization interested in all aspects of the web, sponsors the most extensive set of programs and initiatives devoted to the issue of web accessibility for people with disabilities. W3C’s main umbrella under which these efforts are collected is known as the Web Accessibility Initiative (WAI). “(Carter, 2001)


According to the W3C, the WAI mainly focus on five primary areas of work to pursue web accessibility: (Carter, 2001)


  1. “Addressing accessibility issues in the technology of the Web”

  2. “Creating guidelines for browsers, authoring tools, and content creation”

  3. “Developing evaluation and validation tools for accessibility”

  4. “Conducting education and outreach”

  5. “Tracking research and development”

The WAI has issued many documents for web developers. Following are the three major sets of guidelines in terms of web accessibility: (Carter, 2001)


  1. Web Content Accessibility Guidelines.

  2. Authoring Tool Accessibility Guidelines

  3. User Agent Accessibility Guidelines.

Web Content Accessibility Guidelines are based on two themes: (Carter, 2001)


  1. “Ensuring graceful transformation” (“making Web content accessible and clear despite a person’s disability and the limitations of any hardware of software he or she is using”)

  2. “Making content understandable and navigable” (“by using clear language and navigation so that people with disabilities can quickly and easily orient themselves in a page”).

Authoring Tool Accessibility Guidelines are “addressed to developers of web authoring tools (including web editors, word processors, desktop-publishing software, and software for turning desktop-published documents into HTML), are intended to ensure that authoring tools be accessible to authors regardless of disability, that they produce accessible content by default, and that they assist the author in creating accessible content. “ (Carter, 2001)


User Agent Accessibility Guidelines are “addressed to developers of browsers and other software that presents websites on a user’s screen. A user agent that conforms to these guidelines will be accessible by virtue of its own interface and by virtue of its ability to communicate with assistive technologies. “(Carter, 2001)


The Web Content Accessibility Guidelines 1.0 developed by W3C as part of WAI is the most comprehensive and influential accessibility policy that has been gradually created." (Carter, 2001)


Besides its contribution to the W3C's Web Content Accessibility Guidelines, the Trace Center has also initiated programs (eg. “he Cooperative Electronic Library on Disability) and technology (eg. the touchscreen accessibility feature called EZ Access)” (Carter, 2001)

Tools from CPB/ WGBH National Center

CPB/ WGBH National Center for Accessible Media (NCAM) founded a Web Access Project in 1996. This project “researches, develops and tests methods of integrating access technologies (such as captioning and audio description) and new Web tools into a World Wide Web site, making it fully accessible to blind or deaf Internet users. (Carter, 2001)” The Web Access Project is a leading organization in making multimedia accessible to disabled people. (Carter, 2001)
